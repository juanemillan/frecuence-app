## Implementation Architecture

### Option A: Category filter on existing RoutinesTab
Add a `category` field to the `Routine` interface, add a category filter bar at top of tab. Simplest — no new screens.

```ts
interface Routine {
  // ...existing fields
  category: 'pelvic' | 'morning' | 'facial' | 'breathing' | 'mobility' | 'eyes' | 'core';
}
```

### Option B: New "Programas" tab replacing or alongside RoutinesTab
A dedicated browse UI with category cards, each opening a list of routines in that category. More visual differentiation.

### Option C: Smart Home feed
AI-style "what should I do today" suggestions based on time of day + recent activity. Morning → suggest morning route. Late night → suggest breathing. Most premium feel.

## WorkoutScreen Compatibility Notes

The current `WorkoutScreen` uses a **sets × reps × hold × rest** format which works for:
- ✅ Pelvic floor (hold-based)
- ✅ Core exercises (plank holds)
- ✅ Facial exercises (hold-based)
- ⚠️ Breathing — needs a different timer mode (inhale/exhale phases instead of reps)
- ⚠️ Morning cardio — needs duration-based (30s on / 10s rest) vs reps
- ⚠️ Eye exercises — very short, different cadence

**Recommendation:** Add an `exerciseMode: 'hold' | 'timed' | 'breathing'` field to `RoutineExercise` and conditionally render the appropriate timer UI in `WorkoutScreen`.
